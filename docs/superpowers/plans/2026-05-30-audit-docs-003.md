# AUDIT-DOCS-003 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make README coverage for every MCP tool release-blocking in `npm run audit:docs`, including advanced tools such as `claude_export`.

**Architecture:** Keep the audit as a source-derived consistency check: `scripts/audit-docs.mjs` extracts tool names from `src/server.ts`, compares them with README mentions, and emits failures for missing tool documentation. Update README so each current advanced tool appears in the advanced tools list with a concise purpose.

**Tech Stack:** Node.js ESM audit script, Vitest subprocess tests, Markdown README.

---

## 1. Background And Goal

Current behavior:

- `scripts/audit-docs.mjs` prints `README.md: informational - missing advanced tools: ...` to stdout when README omits a tool present in `BASE_TOOL_DEFINITIONS`.
- `npm run audit:docs` still exits 0 for this condition.
- The current live warning names `claude_export`, so a newly published tool can remain undocumented while release checks pass.

Target behavior:

- Missing README coverage for any source-defined `claude_*` tool must fail `audit:docs`.
- The failure must name each missing tool.
- README must document `claude_export` under advanced/debug tools and include it in the advanced `enabled_tools` example.

Out of scope:

- Do not redesign README structure.
- Do not change MCP tool registration or default enabled tool behavior.
- Do not hand-edit `dist/` or generated plugin server bundles.

## 2. File Impact Scope

`scripts/audit-docs.mjs`
- Change missing advanced tool reporting from stdout informational output to `failures`.
- Keep source-derived extraction from `src/server.ts`.

`tests/audit-docs.test.ts`
- Add a failing fixture proving a README that omits an advanced source-defined tool exits 1.
- Add or update a passing fixture so a README mentioning all source-defined tools exits 0.

`README.md`
- Add `claude_export` to the advanced `enabled_tools` example.
- Add a concise table row describing `claude_export`.

`docs/product/2026-05-30-handoff.md`
- Record `AUDIT-DOCS-003` completion and verification.

Forbidden:

- `dist/` must not be manually edited.
- `plugins/codex-claude-delegate/server/*.js` must not be manually edited.

## 3. Positive Acceptance Cases

Scenario: README documents every source-defined tool.

Input fixture:

```text
src/server.ts contains BASE_TOOL_DEFINITIONS with claude_setup, claude_task, claude_result, claude_apply, claude_cleanup, claude_job_wait, claude_export.
README.md contains each of those tool names.
```

Expected:

- `node scripts/audit-docs.mjs` exits 0.
- stdout contains `doc audit ok`.
- stderr is empty.

Scenario: Real repository README documents `claude_export`.

Expected:

- `README.md` advanced `enabled_tools` example includes `"claude_export"`.
- README advanced/debug tools table includes a row for `claude_export`.
- `npm run audit:docs` exits 0 with no missing advanced tools warning.

## 4. Negative Acceptance Cases

Scenario: README omits an advanced source-defined tool.

Input fixture:

```text
src/server.ts contains BASE_TOOL_DEFINITIONS with claude_export.
README.md omits claude_export but otherwise passes all required phrase/default tool checks.
```

Expected:

- `node scripts/audit-docs.mjs` exits 1.
- stderr contains `README.md: missing advanced tool claude_export`.
- stdout does not contain an informational missing advanced tools line.

Scenario: README mentions unknown `claude_*` tool.

Expected:

- Existing unknown-tool test still exits 1.
- Error remains `README.md: mentions unknown tool <name> not in BASE_TOOL_DEFINITIONS`.

## 5. Forbidden Behaviors

- Do not keep missing advanced tools as informational warnings.
- Do not hard-code only `claude_export`; the audit must derive missing tools from `BASE_TOOL_DEFINITIONS`.
- Do not require advanced tools to be in the default enabled tools list.
- Do not fail when README documents `claude_delegate` as the MCP server name.
- Do not change default enabled tool count from 5.
